Please read the requirements document located at `docs/requirement.md` in the workspace. Based on the requirements specified in that document, design a comprehensive data schema for storing application data in the web browser's local storage.

The data schema should include:
1. Clear structure for each data entity/object that needs to be stored
2. Field names, data types, and descriptions for each property
3. Any relationships between different data entities
4. Constraints or validation rules where applicable
5. Considerations for data serialization (since local storage stores strings)
6. Any indexing or key strategies for efficient data retrieval

Output the complete data schema design to a new file at `docs/data-schema.md`. The documentation should be well-structured, easy to understand, and include examples where helpful.
