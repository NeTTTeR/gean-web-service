Please read and analyze the requirements document located at `docs/requirement.md` and create a comprehensive UX/UI requirements document that will serve as the foundation for the UI/UX design team to create screen design prototypes and mockups.

The UX/UI requirements document should include the following sections:

1. **Complete User Flow Diagram**: 
   - Provide a clear textual representation (using Mermaid diagram syntax or structured text) showing how users navigate through all screens/pages in the application
   - Include entry points, decision points, and transitions between screens
   - Show the complete user journey from initial access to task completion
   - Indicate any conditional flows or alternative paths

2. **Screen-by-Screen Breakdown**: 
   For each screen/page identified in the user flow, document the following:
   - **Screen Name**: Clear, descriptive name for the screen
   - **Screen Purpose**: Brief description of what the screen accomplishes and its role in the user journey
   - **User Interactions**: List all interactive elements and actions available (e.g., buttons, links, tabs, dropdowns, gestures)
   - **Input Fields**: For each input field, specify:
     * Field name/label
     * Field type (text, email, password, number, date, dropdown, checkbox, radio button, file upload, etc.)
     * Validation rules (format requirements, min/max length, allowed characters, etc.)
     * Whether the field is required or optional
     * Any default values or placeholder text
     * Any dependencies on other fields
   - **Output/Displayed Data**: Document what information is shown to users:
     * Data elements displayed on the screen
     * Data format and structure
     * Dynamic vs. static content
     * Any conditional display logic
   - **Navigation**: Specify how users can navigate to/from this screen
   - **Error States**: Describe error messages and validation feedback that should be displayed
   - **Success States**: Describe confirmation messages or success indicators

**Exclusions**: 
The document should NOT include visual design specifications such as:
- Layout, positioning, or spacing details
- Color schemes, typography, or styling
- Visual design patterns or UI component libraries
- Wireframes or mockups

These visual design elements will be created by the UI/UX design team based on this requirements document.

**Output**: 
Save the completed UX/UI requirements document to `docs/ui-requirement.md`. The document should be:
- Well-structured with clear headings and sections
- Written in Markdown format for easy readability
- Comprehensive enough for designers to create high-fidelity mockups without ambiguity
- Focused on functionality and user interactions rather than visual appearance
